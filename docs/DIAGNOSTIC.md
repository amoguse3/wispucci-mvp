# Adaptive Diagnostic

## Job

In ~10 minutes (10-15 items) produce a **mastery map over the whole graph**
without asking one question per node. Then hand the tutor a prioritized
learning path. This is how the bot 'determines the student's skill' instead
of guessing.

## Policy (`diagnostic.py`)

**1. Max-information item selection.** Always ask the skill whose mastery is
closest to 0.5 — a coin-flip is where we learn the most — scaled by
`exam_weight` so questions are spent where the exam spends points. A repeat
penalty discourages hammering one skill.

**2. Prerequisite propagation.** A confident *correct* answer on a deep skill
soft-credits its prerequisites (capped at 0.85 — inference, not proof), so we
don't waste items re-testing foundations we can infer are solid. A *miss*
pushes probes down toward prerequisites to locate the true gap.

**3. Early stop.** Stop when enough skills are 'confident' (mastery outside
the `[0.35, 0.65]` uncertainty band) or `max_items` is reached.

## Output (`Diagnostic.report()`)

```json
{
  "items_used": 13,
  "exam_readiness": 0.61,
  "mastery": { "radicali": 0.42, "ecuatii_gr2": 0.28, ... },
  "weak_topics": [ {"skill": "ecuatii_gr2", "title": "...", "mastery": 0.28} ],
  "recommended_path": [ {"skill": "radicali", "title": "..."}, ... ]
}
```

`weak_topics` is exam-risk-ranked; `recommended_path` respects prerequisites
and front-loads high-yield skills.

## Question source

The diagnostic is the **policy**, not the item bank. Items are generated by
`ai_tutor.generate_exam()` grounded in the KB, or pulled from tagged past
ANCE items. Each item must carry its `skill_id` and `difficulty` so results
feed straight back into `StudentModel.observe()`.

## Loop

```python
diag = Diagnostic(model)
while (item := diag.next_item()) is not None:
    # generate/serve a question for item.skill_id at item.difficulty
    correct, confidence, error_type = serve_and_grade(item)
    diag.record(item.skill_id, correct, confidence=confidence, error_type=error_type)
report = diag.report()
```
